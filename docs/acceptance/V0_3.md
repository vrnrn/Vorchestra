# v0.3 acceptance record

## Status

The v0.3 implementation is complete in the current worktree. Automated engine,
runner, desktop, disposable-Git, and production Electron smoke checks pass.
Authenticated real-runtime execution remains a separate opt-in activity because
it can consume account usage, access a network, and modify files.

An authenticated Cline plan-mode smoke was explicitly authorized and completed
after the original implementation pass. No filesystem tools were requested or
used.

This record proves Vorchestra-owned contracts and integration behavior. It does
not claim that every locally installed runtime is authenticated, healthy, or
able to reach its provider at the time a workflow is opened.

## Architecture evidence

- `packages/engine` contains generic process arguments, deterministic templates,
  artifact routing, DAG scheduling, and runtime events. It contains no Codex,
  Cline, Antigravity, model-provider, or Git-worktree scheduling concepts.
- `apps/desktop/src/shared/agent-runtime.ts` owns runtime identities,
  capabilities, editor reconstruction, and compilation to generic process
  definitions.
- `apps/desktop/src/main/agent-worktrees.ts` and `worktree-runtime.ts` own
  explicit Git preparation and lifecycle. Participating process blocks receive
  only a resolved generic working directory.
- Runtime authentication, provider credentials, and private sessions remain
  owned by the selected local CLI. The model selector is populated only from the
  user's local Vorchestra settings file.
- Worktree changes, diffs, commits, and runtime events never become implicit
  workflow artifacts.

## Safety evidence

- Direct execution and `shell: false` remain the generated default.
- Codex, Cline, and Antigravity compiler tests reject known approval,
  permission, and sandbox bypass flags. Codex includes `--skip-git-repo-check`
  so explicitly reviewed workflows can run in ordinary non-Git directories; this
  does not change the selected Codex sandbox.
- Read-only remains the default editor choice. Cline's visible `--plan` mapping
  is behavioral intent rather than a hard sandbox, and the product states this
  limitation explicitly.
- Instructions, templates, model overrides, runtime identity, working directory,
  repository root, base ref, worktree scope, and compiled invocation are visible
  before execution.
- Changed, failed, cancelled, artifact-dependent, or inspection-failed worktrees
  are retained. Automatic cleanup is restricted to unchanged scopes; the desktop
  never commits, merges, rebases, pushes, opens a pull request, or discards
  changes.

## Real-runtime readiness

The Antigravity contract was checked against installed `agy` 1.0.15 and its
local help output. The Cline contract was checked against the repaired installed
Cline 3.0.39 CLI, its local help output, and the separately inspected Kanban
adapter as a non-authoritative reference. An authenticated JSON plan-mode smoke
completed in 3.0 seconds with exit code 0, no tool calls, and the exact
requested response. A second smoke using Vorchestra's exact generated argument
shape, `cline --plan <prompt>`, returned `VORCHESTRA_CLINE_CONTRACT_OK` with
exit code 0.

Cline documents auto-approval as enabled by default in act mode. Vorchestra's
validated read-only mapping uses visible `--plan` behavior and does not generate
an approval-bypass flag. This acceptance pass does not claim a hard filesystem
sandbox for Cline plan mode and did not exercise a workspace-write Cline run.

Follow-up connected-context validation found that Cline 3.0.39 reads piped stdin
in `--json` mode but ignores it in ordinary text-output mode. Vorchestra now
marks separate Cline stdin context unsupported, composes connected text into the
visible deterministic positional template, and migrates legacy v0.3 Cline stdin
blocks before preflight and execution. This keeps downstream response artifacts
as clean text rather than exposing Cline's JSONL event stream.

The Agent model selector loads its per-runtime choices and optional defaults
from the user-owned `~/.vorchestra/models.json`. The application contains no
compiled provider model list. A configured default is applied only when a block
has no explicit model; listed alternatives and an exact Custom value remain
available in the editor. Invalid settings are preserved, reported actionably,
and produce an empty selector catalog rather than guessed model identifiers.

Follow-up Antigravity 1.1.1 validation proved that interactive launch respects
cwd but headless `agy --print` selects the global scratch workspace. Vorchestra
now passes the final resolved working directory through `--add-dir`, including
the rewritten path for workflow-run worktrees. This keeps writes in the visible
scope without `--new-project` or persistent Antigravity project creation.

`multi-runtime-worktree.vorchestra.json` is the explicit opt-in smoke workflow
for Codex, Cline, and Antigravity. It uses a disposable run-scoped worktree,
orders all three Agents through text artifacts, uses a deterministic template
for the Antigravity handoff, and declares every generated file. It must be run
only after reviewing local authentication, model defaults, repository choice,
and account usage.

No fresh VoiceOver walkthrough was required for this implementation pass. The
existing accessibility tests remain green, and the new controls use ordinary
labelled inputs and buttons covered by the desktop test suite.
