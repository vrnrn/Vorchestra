# v0.2 acceptance record

## Status

The v0.2 implementation and release acceptance are complete in the current
worktree. The release is intentionally unsigned and unnotarized. Automated
artifact checks, packaged runtime and performance checks, and a packaged
keyboard and VoiceOver walkthrough have passed on supported Apple silicon
hardware.

This record maps every release target to authoritative implementation evidence.
It does not present checksums, an unsigned package, or jsdom tests as evidence
of Apple publisher verification or malware review.

## Automated gates

The following gates pass on 2026-07-09:

```sh
npm run verify
npm run build
npm run desktop:smoke
npm run desktop:smoke:packaged
npm run desktop:performance:packaged
npm run release:mac
npm run workflow:check -- examples/workflows/text-to-file.vorchestra.json
```

`npm run verify` currently covers 41 engine tests, 27 Node-runner tests, and 88
desktop tests.

The trusted shell-free example was also executed from an isolated temporary
directory with `text-to-file.example.json`: `tee` created a 24-byte file, the
filesystem-reference artifact reached the downstream `wc` block, and both blocks
succeeded. The temporary output was removed after verification.

The opt-in `codex-file-agent.vorchestra.json` workflow was executed on
2026-07-09 through the real engine and Node runner with authenticated Codex CLI
0.144.1. The exact workspace-write invocation received separate stdin context,
created only the declared `generated/report.md` in the temporary workflow
directory, exposed its filesystem reference, and started the downstream `wc`
block. The file contained exactly one 34-byte line and the temporary workspace
was removed. Packaged UI smoke and compiler tests separately prove the AI Agent
editor and compiled contract without invoking Codex.

## Packaged accessibility walkthrough

On 2026-07-09, the official unsigned Apple silicon 0.2.0 packaged application
was exercised on macOS 27.0 (build 26A5368g), arm64, with VoiceOver 10 enabled
and an isolated disposable user-data profile. The walkthrough covered named and
disabled toolbar controls; Process and AI Agent creation and configuration;
keyboard reordering of arguments, input ports, output ports, and environment
entries; Configure/Run details tab-list behavior; authority-review focus entry,
wrapping, Escape closure, and focus restoration; a missing-input preflight
blocker followed by a supplied manual input; a successful stdout-producing run;
a keyboard-cancelled 60-second run; retained succeeded, cancelled, and failed
history; a filesystem-reference failure with responsible-field navigation; and a
corrected filesystem-reference run exposing Copy and Reveal actions.

VoiceOver was disabled again and its system setting was verified off. The
packaged application exited normally and the isolated profile was removed. The
user's recovered unsaved 51-block draft was never opened or changed.

## v0.2 scenario audit

Automated evidence proves creation of reusable schema-v2 workflows, exact
invocation review, manual typed inputs, non-executing preflight, consistent
workflow-relative path resolution, shell-free file creation, filesystem artifact
inspection and routing, AI Agent compilation and fake generated-file routing,
workspace-write declaration, typed failure diagnosis and field navigation,
corrected reruns, post-restart retained history, editing recovery, ordering
persistence, and portable save/load migration.

Automated tests use an injected fake process runner and never execute the user's
real Codex CLI, network calls, or ambient shell commands.

## Remaining release blockers

None. Fresh-account or fresh-machine installation and removal exercises are not
part of the v0.2 acceptance contract. The documented per-application macOS
override remains user guidance for the intentionally unsigned distribution, not
a claim that application code can or should suppress Gatekeeper.
